## Ops

Operator-style functions. Sometimes useful with higher-order functions. Like with regular JS operators, beware of implicit type coercions.

---

### `add(a, b)`

Same as `+`.

```js
add(1, 2)
// 1 + 2 = 3
```

---

### `sub(a, b)`

Same as `-`.

```js
sub(2, 1)
// 2 - 1 = 1
```

---

### `mul(a, b)`

Same as `*`.

```js
mul(10, 20)
// 10 * 20 = 200
```

---

### `div(a, b)`

Same as `/`.

```js
div(10, 20)
// 10 / 20 = 0.5
```

---

### `rem(a, b)`

Same as `%`.

```js
rem(2.5, 1)
// 2.5 % 1 = 0.5
```

---

### `lt(a, b)`

Same as `<`.

```js
lt(10, 20)
// 10 < 20 = true
```

---

### `gt(a, b)`

Same as `>`.

```js
gt(10, 20)
// 10 > 20 = false
```

---

### `lte(a, b)`

Same as `<=`.

```js
lte(10, 20)
// 10 <= 20 = true
lte(10, 10)
// 10 <= 10 = true
```

---

### `gte(a, b)`

Same as `>=`.

```js
gte(10, 20)
// 10 >= 20 = false
gte(10, 10)
// 10 >= 10 = true
```

---

### `inc(num)`

Increments by `1`.

```js
inc(1)
// 1 + 1 = 2
```

---

### `dec(num)`

Decrements by `1`.

```js
dec(2)
// 2 - 1 = 1
```
